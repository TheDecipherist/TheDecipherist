# 01 — TheDecipherist GitHub Pages Site

## Purpose

Build a world-class static GitHub Pages portfolio for TheDecipherist (Tim Carter Clausen) — a Danish-American full-stack architect, developer tool builder, and cryptographic researcher. The site serves a technical audience (developers, engineers, AI tool builders, MongoDB/DevOps professionals) as the primary portfolio, and a secondary audience (cipher enthusiasts, true crime community) via a separate `/cipher/` section. Every line of CSS is hand-authored — zero frameworks, zero utility classes, zero hex colors.

## Architecture

Pure static HTML/CSS/JS site served from the `/docs/` folder via GitHub Pages on branch `main`. No build system required — one shell script concatenates CSS source files into a single production `main.css`. Images served from `/docs/images/` with full AVIF/WebP/JPEG responsive variants once generated.

```
/docs/                 ← GitHub Pages root (served)
  index.html           ← Main technical portfolio (homepage)
  cipher/index.html    ← Cipher work hub (separate world)
  articles/index.html  ← Articles index
  projects/index.html  ← Projects page
  resume/index.html    ← Résumé page
  css/
    main.css           ← Single concatenated production stylesheet (built by scripts/build-css.sh)
    src/               ← 16 source CSS files organized by layer
  js/                  ← 4 JS modules (ES module, Houdini worklet, Three.js particles, prefetch)
  fonts/               ← Self-hosted JetBrains Mono + Clash Display (woff2, Latin subset)
  images/              ← Responsive AVIF/WebP/JPEG variants (generated by scripts/convert-images.sh)
  icons/               ← favicon.svg, icon-192.png, icon-512.png, apple-touch-icon.png
  site.webmanifest
  CNAME                ← thedecipherist.com

/scripts/              ← Build tooling (repo root, NOT served)
  build-css.sh         ← Cat source CSS → docs/css/main.css
  convert-images.sh    ← Squoosh-based AVIF/WebP/JPEG responsive conversion
```

Content source of truth: `./README.md` (stats, bio, articles, projects) and `./articles/` (individual article dirs with markdown + webp images).

## Business Rules

**CSS rules (non-negotiable):**
- Zero hex values — all colors use OKLCH
- Zero Tailwind / Bootstrap / utility framework classes
- BEM naming convention throughout — `.hero__headline`, `.article-card`, `.stat-counter`
- All gradients use `in oklch` interpolation
- @property registered for every animated custom property
- Section dividers use `clip-path: shape()` — not SVG, not border
- Gradient borders via pseudo-element mask (never solid borders)
- Native scroll-driven animations via `animation-timeline: view()` (no Intersection Observer)
- CSS `if()` for `prefers-reduced-motion` transitions
- `@starting-style` for panel/detail entries
- Native CSS masonry grid (`grid-template-rows: masonry`) on projects section
- `sibling-index()` for card stagger delays
- `size-adjust` on both `@font-face` declarations
- Cross-document `@view-transition` on every HTML file

**HTML rules:**
- Zero `<div>` where a semantic element exists
- `role="list"` on every `<ul>` or `<ol>` with `list-style: none`
- `<time datetime="YYYY-MM-DD">` on every date
- `<mark>` for SOLVED badges
- `<dl>/<dt>/<dd>` for key-value metadata
- `rel="me"` on all social links
- `lang="en"` on every `<html>`
- Explicit `width` and `height` on every `<img>`
- Skip link as first body element

**Performance rules:**
- Critical CSS inlined in every `<head>` — zero external CSS on first paint
- Max 4 `<link rel="preload">` per page
- Max 2 `<link rel="preconnect">`
- LCP image: `fetchpriority="high"`, `loading="eager"`, `decoding="sync"`
- Three.js loads only on `window` load event
- GSAP loads with `defer`

**Image rules (for this build):**
- No images exist yet — use CSS gradient fallbacks on all `<picture>` elements
- No placeholder services (no picsum, no placehold.it)
- Each image placeholder uses a CSS gradient fallback element
- Cipher work articles go to `/cipher/` ONLY — never on main index

**Content rules:**
- All stats, titles, descriptions sourced from `./README.md` exactly — no invented content
- Cipher/research articles (Zodiac, Voynich, Beale, March Impromptu, Baber) go to `/cipher/` ONLY
